class Data_Fusion.Data_Fusion(cwd: str = '.', Inventory_name: str = 'UCLA_Inv')

A class for managing, downloading, and fusing geospatial building inventory datasets, including Microsoft, FEMA, NSI, and imputed bounding boxes, in the U.S., for now.

Initialization of the Inventory_Loss object involves:

Parameters:

  • cwd (str) – User-defined directory to store outcomes. Defaults to ‘.’, which stores results in the directory where the code is executed.

  • Inventory_name (str) – Inventory name, defined by the user, to label the main directory where inventory data are stored.

  • Download (bool) – If set to True, downloads a sample dataset to run the loss model and perform integrity tests.

Variables:

  • cwd (str) – Path to the working directory (access through Data_Fusion(object).cwd).

  • county_bound (GeoDataFrame) – GeoDataFrame of U.S. county boundaries with FIPS codes (access through Data_Fusion(object).county_bound).

  • us_state_to_abbrev (dict) – Dictionary mapping state names to abbreviations (access through Data_Fusion(object).us_state_to_abbrev).

  • utils (class) – Reference to the helper utils class (access through Data_Fusion(object).utils).

  • state_codes (dict) – Dictionary mapping state abbreviations to FIPS codes (access through Data_Fusion(object).state_codes).

Returns:

None

Return type:

None

Example

>>> Inv = Data_Fusion("C:/Users/Sam/Documents","UCLA_Inv_1")
Microsoft(State: str, Save_County: bool = True)

Download and organize Microsoft building footprint data for a specified U.S. state (see Directory Structure to get familiar with where and how the data is stored).

The data is retrieved from the Microsoft USBuildingFootprints repository and stored locally in state- and county-specific folders.

Microsoft hosts the data at: https://minedbuildings.z5.web.core.windows.net/legacy/usbuildings-v2/

Parameters:

  • State (str) – Full name of the U.S. state (e.g., “California”).

  • Save_County (bool, optional) – Whether to save outputs separately for each county. Defaults to True.

Returns:

None

Example

>>> Data_Fusion.Microsoft("Texas")
FEMA(State: str, Save_County: bool = True)

Download and extract FEMA building footprint data for a specified U.S. state (see Directory Structure to get familiar with where and how the data is stored).

This method queries the FEMA Structures Dataset, downloads the ZIP archive containing the GDB file, extracts it, and loads it into a GeoDataFrame. The output is stored in state- and county-specific folders, and the data can optionally be split by county.

Parameters:

  • State (str) – Full name of the U.S. state (e.g., “Florida”).

  • Save_County (bool, optional) – Whether to save outputs for each county individually. Defaults to True.

Returns:

None

Example

>>> Data_Fusion.FEMA("Vermont")
NSI(State: str, Save_County: bool = True)

Download and organize NSI (National Structure Inventory) 2022 data for a specified U.S. state (see Directory Structure to get familiar with where and how the data is stored).

centroid_sdjoin(A: GeoDataFrame, B: GeoDataFrame) → DataFrame

Perform a symmetric spatial join between two GeoDataFrames based on centroid containment.

This method identifies spatial relationships where: - The centroid of a geometry in A lies within a geometry in B - The centroid of a geometry in B lies within a geometry in A

It returns a combined DataFrame of matched feature indices from both join directions.

Parameters:

  • A (GeoDataFrame) – First GeoDataFrame.

  • B (GeoDataFrame) – Second GeoDataFrame.

Returns:

A combined DataFrame with two specific columns:

  • index_A: Macthed index from GeoDataFrame A.

  • index_B: Macthed index from GeoDataFrame B.

to trace the macthed indices.

DataFrame

Algorithm3(F: GeoDataFrame, gdN: GeoDataFrame, gdM: GeoDataFrame, Stat: list[str], Mc: list[str], Fc: list[str], Nc: list[str])

Match NSI point data to fused building footprints (Microsoft + FEMA) using centroid-based spatial matching. Details availabe in the documentation is available at: Data_Fusion Details

This function expands NSI matches based on two strategies: 1. Direct matching between NSI and Microsoft-FEMA fused polygons. 2. Aided matching based on the imputed bounding boxes.

It merges NSI attributes to building polygons and updates summary statistics related to NSI coverage.

Parameters:

  • F (GeoDataFrame) – Input fused polygon layer (from Algorithm 2) with ‘geometry_F’ and ‘geometry_M’ columns.

  • gdN (GeoDataFrame) – NSI point dataset with ‘ID_N’ and ‘geometry_N’ columns.

  • gdM (GeoDataFrame) – Original Microsoft polygons (used for NSI coverage stats).

  • Stat (list) – A mutable list to which NSI matching statistics will be appended.

  • Mc (list[str]) – List of Microsoft column names to retain.

  • Fc (list[str]) – List of FEMA column names to retain.

  • Nc (list[str]) – List of NSI column names to retain.

Returns:

  • M_F_N (GeoDataFrame): Updated fusion layer with NSI attributes and match types.

  • Stat (list[float]): NSI matching statISTICS (See documentation at Data_Fusion Details).

Return type:

Tuple[GeoDataFrame, list[float]]

match_data_county(gd: GeoDataFrame, database: str, gd_C, State_Ab: str, Save_County: bool = True)

Generate Open Location Codes (also known as Plus Codes) for a GeoDataFrame of point or polygon geometries.

The method computes OLCs for the centroid of each geometry using the Google Open Location Code library. If duplicate codes are found, they are made unique by appending a numeric suffix (e.g., _1, _2).

Parameters:

  • gd (GeoDataFrame) – A GeoDataFrame of points or polygons. The centroid is used if geometry is a polygon.

  • database (str) – The name tag that user specifies to save data, e.g., Microsoft

  • gd_C (GeoDataFrame) – The U.S. county border shapefile.

  • State_Ab (str) – The Satte abbrevaited name.

  • Save_County (bool) – User’s choice to save outputs on county level or not.

Returns:

None

col_names(gd, Mc='', Fc='', Nc='', suffix=['_x', '_y'], extra='')

Harmonize and extract relevant column names from a merged GeoDataFrame.

This method resolves column name conflicts caused by merge or sjoin operations by renaming columns with suffixes (e.g., “_x”, “_y”) back to their original names. It collects columns from Microsoft, FEMA, and NSI datasets as specified by the user, along with any additional columns.

Parameters:

  • gd (GeoDataFrame) – The merged GeoDataFrame to process.

  • Mc (List[str] or str, optional) – List of Microsoft columns to retain and rename if needed.

  • Fc (List[str] or str, optional) – List of FEMA columns to retain and rename if needed.

  • Nc (List[str] or str, optional) – List of NSI columns to retain and rename if needed.

  • suffix (List[str], optional) – Suffixes to remove from merged columns (e.g., [‘_x’, ‘_y’] or [‘_left’, ‘_right’]).

  • extra (List[str] or str, optional) – Additional columns to retain, such as ‘Type’ or ‘group_id’.

Returns:

A subset of the original GeoDataFrame with standardized and selected columns.

Return type:

GeoDataFrame

M_F_N_Fusion(State: str, Coun_ty: str = '')

Perform spatial and point-level geodatabase fusion for Microsoft, FEMA, NSI, and imputed bounding boxes by processing the data and applying Algorithm 2 and Algorithm 3 (see documentation at Data_Fusion Details).

Parameters:

  • State (str) – U.S. state name (e.g., “Texas”).

  • Coun_ty (list[str], optional) – List of county FIPS codes. If empty, all counties in the state are processed.

Returns:

None

This method does not return any value, but populates the “Transitory data” directory with the fusion results (see directory).

Example

>>> Data_Fusion.M_F_N_Fusion("Texas")
>>> Data_Fusion.M_F_N_Fusion("California", Coun_ty=["06037"])  # Los Angeles County only
